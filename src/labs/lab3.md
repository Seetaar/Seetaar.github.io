# Лабораторная работа №3: CI/CD для статического сайта в SourceCraft

## Цель работы

- Настроить процесс CI/CD для статического сайта. Добавить репозиторий на SourceCraft

## Ход Работы

1. Создание репозитория на SourceCraft
 
2. Настройка CI/CD для автоматизации

    Для автоматизации процесса сборки и публикации были созданы конфигурационные файлы в директориях .sourcecraft/ и .github/ (был добавлен позже). Workflow автоматически запускается при каждом push в ветку main.
3. Перенос данных

    Для полного переноса истории коммитов и всех веток использована команда push --mirror
4. Развертывание сайта
    
    На платформе SourceCraft автоматически запустился CI/CD-пайплайн.
Куб build-docs:
Загружен образ python:3.14-slim.
Установлены зависимости из requirements.txt.
Сгенерированы статические HTML-файлы командой mkdocs build.
Куб publish-site:
Создана (или обновлена) ветка release.
Содержимое папки ./site скопировано в корень ветки.
Изменения закоммичены и отправлены в удалённый репозиторий.
Файл .sourcecraft/sites.yaml связал ветку release с веб-хостингом.
5. Итог

    Сайт успешно открыт по адресу: https://seetaar.sourcecraft.site/lab-3/.


## Вывод
В ходе выполнения лабораторной работы №3 был успешно осуществлен перенос веб-сайта с платформы GitHub Pages на платформу SourceCraft.
